# План обучения: Тестирование микросервисов

## Этап 1: Фундаментальные основы (2 недели)

### Неделя 1: Концепции и стратегия
- **Понимание микросервисной архитектуры** - отличия от монолитных приложений
- **Пирамида тестирования** - адаптация для distributed systems
- **Типы тестов** - unit, integration, contract, e2e и их роль в микросервисах
- **Изоляция и зависимости** - проблемы network latency и partial failures

### Неделя 2: Основы изоляции
- **Test doubles** - mocks, stubs, fakes в контексте микросервисов
- **Тестовые данные** - управление состоянием в distributed environment
- **Test fixtures** - настройка окружения для независимых тестов
- **Async тестирование** - работа с событиями и message queues

## Этап 2: Unit и Integration тестирование (3 недели)

### Неделя 3: Качественные unit тесты
- **Архитектура для тестируемости** - dependency injection, clean architecture
- **Тестирование бизнес-логики** - изоляция от внешних зависимостей
- **Edge cases и error handling** - покрытие граничных условий
- **Performance unit тестов** - быстрая обратная связь

### Неделя 4: Integration стратегии
- **TestContainers** - тестирование с реальными зависимостями
- **In-memory alternatives** - когда использовать embedded databases
- **API testing** - REST, GraphQL, gRPC endpoints
- **Database integration** - миграции, транзакции, connection pooling

### Неделя 5: Component тестирование
- **Service-level тесты** - тестирование сервиса как черного ящика
- **Test slices** - фокусированное тестирование слоев
- **Configuration testing** - проверка настроек и environment variables
- **Health checks и readiness probes**

## Этап 3: Contract Testing (2 недели)

### Неделя 6: Consumer-Driven Contracts
- **Pact framework** - создание и верификация контрактов
- **API versioning** - обратная совместимость и эволюция
- **Schema validation** - JSON Schema, OpenAPI, Protocol Buffers
- **Contract evolution** - безопасные изменения API

### Неделя 7: Advanced Contract Testing
- **Provider testing** - верификация на стороне поставщика
- **Contract management** - Pact Broker, version control
- **GraphQL contracts** - специфика тестирования GraphQL API
- **Event-driven contracts** - тестирование async messaging

## Этап 4: End-to-End тестирование (2 недели)

### Неделя 8: E2E стратегия
- **Test environment management** - изоляция тестовых сред
- **User journey mapping** - критические пути пользователя
- **Data management** - создание и очистка тестовых данных
- **Cross-browser и mobile testing**

### Неделя 9: Управление E2E тестами
- **Flaky tests prevention** - стабильность и надежность
- **Parallel execution** - масштабирование E2E тестов
- **Visual regression testing** - UI consistency
- **Accessibility testing** - соответствие стандартам доступности
